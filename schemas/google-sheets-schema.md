# Google Sheets Schema

Patient Retention Engine — Sheet Definitions

---

## Overview: New Data Flow

```
7:00 AM Daily
     │
     ▼
WF9 creates a new Google Sheet named:
"Patient Intake - YYYY-MM-DD"
  + emails the link to the clinic
     │
     ▼
Clinic staff fills in patient rows
(Columns B–J only — rest are auto-filled)
     │
     ▼ (every 10 minutes)
WF10 polls the sheet, validates rows,
maps fields → upserts into Supabase
     │
     ▼
Supabase = SOURCE OF TRUTH
     │
     ▼
WF1–WF8 read ONLY from Supabase
```

**Google Sheets role:** Input + daily intake form only (write by staff, sync by WF10).
**Supabase role:** Persistent source of truth, powering all automation workflows.

## Sheet 1: Daily Intake Sheet (Auto-Created Each Morning)

A **new spreadsheet** is created every morning at 7 AM by WF9. The spreadsheet is named:

```
Patient Intake - YYYY-MM-DD
```

e.g. `Patient Intake - 2026-04-15`

It is placed in the Google Drive folder configured in `GOOGLE_DRIVE_FOLDER_ID`.

### Column Definitions

#### A — `id`
- Format: `PAT-0001`, `PAT-0002` …  
- Auto-generated by WF10 on sync  
- Left blank by staff  
- Column is protected (editors cannot modify)  
- n8n idempotency key: rows with a non-empty `id` are **skipped** by WF10 (already processed)

#### B — `hospital_name`
- Dropdown (data validation)  
- Values: configured in `CLINIC_HOSPITAL_NAMES` env var  
- Default placeholders: `City Hospital`, `General Hospital`, `Metro Clinic`  
- **Required** — rows without this value are flagged as `data_error`

#### C — `doctor_name`
- Dropdown (data validation)  
- Values: configured in `CLINIC_DOCTOR_NAMES` env var  
- Default placeholders: `Dr. Sharma`, `Dr. Mehta`, `Dr. Patel`, `Dr. Kumar`  
- **Required**

#### D — `patient_name`
- Free text, min 2 characters  
- **Required**

#### E — `dob`
- Date format: `DD/MM/YYYY` (Google Sheets date picker)  
- Stored in Supabase as `DATE`  
- Must be in the past if provided  
- Optional

#### F — `sex`
- Dropdown: `Male`, `Female`, `Other`  
- Optional

#### G — `phone_number`
- **Text column** (not numeric — formatted as Plain Text)  
- 10 digits, no country code, no spaces, no dashes  
- WF10 prepends `+91` to form E.164 format for WhatsApp  
- Regex validated: `^[0-9]{10}$`  
- **Required** — used as the WhatsApp communication key and Supabase unique key

#### H — `visit_date`
- Date format: `DD/MM/YYYY`  
- Must not be in the future  
- **Required**

#### I — `follow_up_required`
- Dropdown: `Yes`, `No`  
- **Required** — drives follow-up automation logic  
- If `Yes` → `follow_up_date` (column J) must be filled  
- If `No` → `follow_up_date` must be empty

#### J — `follow_up_date`
- Date format: `DD/MM/YYYY`  
- **Conditional required:** must be filled if `follow_up_required = Yes`, must be empty if `No`  
- Must be strictly after `visit_date`  
- Read by WF1 (day-before reminder), WF2 (same-day), WF3 (missed recovery)

#### K — `status`
- Dropdown: `Pending`, `Completed`, `Missed`  
- **Auto-set to `Pending`** by WF10 on sync  
- Staff can manually update to `Completed` when patient attends  
- WF3 automatically updates to `Missed` after 7 days overdue  
- Color coding: Pending → Yellow, Completed → Green, Missed → Red

#### L — `created_at`
- ISO 8601 timestamp: `2026-04-15T07:00:00.000Z`  
- Auto-filled by WF10 when row is first synced  
- Protected (read-only for staff)

#### M — `updated_at`
- ISO 8601 timestamp  
- Auto-updated by WF10 and n8n workflows on state change  
- Protected (read-only for staff)

### Validation Rules (Applied by WF9 via Google Sheets API)

| Column | Rule | Error Behaviour |
|--------|------|-----------------|
| B — hospital_name | Dropdown: values from `CLINIC_HOSPITAL_NAMES` | Reject input |
| C — doctor_name | Dropdown: values from `CLINIC_DOCTOR_NAMES` | Reject input |
| F — sex | Dropdown: Male, Female, Other | Reject input |
| G — phone_number | Custom formula: `=AND(LEN(G2)=10, ISNUMBER(VALUE(G2)))` | Show warning |
| H — visit_date | Date ≤ today | Show warning |
| I — follow_up_required | Dropdown: Yes, No | Reject input |
| J — follow_up_date | Date (when filled) | Show warning |
| K — status | Dropdown: Pending, Completed, Missed | Reject input |

### Example Invalid Rows (WF10 writes error back)

| Scenario | What WF10 Does |
|----------|----------------|
| `phone_number` = `98765` (5 digits) | Writes `id = INVALID`, `status = data_error`, logs error |
| `follow_up_required = Yes` but `follow_up_date` blank | Writes `id = INVALID`, `status = data_error` |
| `follow_up_date` < `visit_date` | Writes `id = INVALID`, `status = data_error` |
| `patient_name` blank | Writes `id = INVALID`, `status = data_error` |
| Duplicate `phone_number` (second occurrence in same sheet) | Second row: `id = DUPLICATE`, `status = data_error` |

---

### Edge Case Handling

| Edge Case | System Response |
|-----------|-----------------|
| Phone already exists in Supabase (returning patient) | Upsert: updates visit_date, follow_up_date, doctor; resets message_count and status to `pending` |
| Staff submits row with no data (empty row) | WF10 detects all required fields empty, skips silently |
| Staff edits a row after `id` is filled | WF10 ignores (id not empty = already processed). Staff must contact admin to reprocess |
| WF9 hasn't run (sheet not created yet) | WF10 queries Supabase, finds no sheet for today, logs WARN, exits cleanly |
| Network error during sheet creation | WF9 catches error, logs to Supabase system_logs, sends admin WhatsApp alert |
| Two staff members submit same phone same day | Only first submission is processed (second row gets `DUPLICATE` id) |

## Legacy Sheet: `Patients` (Master Sheet — DEPRECATED for intake)

The original master sheet (`GOOGLE_SHEETS_ID` in `.env`) is **no longer used for patient intake**.
It may be kept as a read-only archive or removed. All new patients enter via the daily intake sheet.

WF6 (Feedback Listener) reads from Supabase directly — no longer uses the master sheet.

The Google Sheets `Message_Logs` tab is no longer written by any workflow.

## Google Sheets Formatting (Applied by WF9 Automatically)

WF9 applies all of the following to each new daily sheet via the Sheets API batchUpdate:

1. **Freeze Row 1** — header row stays visible while scrolling
2. **Bold + Background colour** on header row (dark blue, white text)
3. **Column B** — Dropdown for `hospital_name` (values from env)
4. **Column C** — Dropdown for `doctor_name` (values from env)
5. **Column F** — Dropdown: `Male`, `Female`, `Other`
6. **Column I** — Dropdown: `Yes`, `No`
7. **Column K** — Dropdown: `Pending`, `Completed`, `Missed`
8. **Conditional formatting on Column K:**
   - `Pending` → Yellow background (`#FFF2CC`)
   - `Completed` → Green background (`#D9EAD3`)
   - `Missed` → Red background (`#F4CCCC`)
9. **Conditional formatting on Column G (phone):**
   - Highlight in orange if length ≠ 10 digits
10. **Protected ranges:**
    - Column A (`id`) — WF10 write-back only
    - Column K (`status`) — WF10 write-back only (staff CAN manually set to Completed)
    - Column L (`created_at`) — read-only
    - Column M (`updated_at`) — read-only
11. **Column G** — formatted as Plain Text (prevents number conversion)

## n8n Node Integration Reference

### WF9 — Daily Sheet Creator
- **Node type:** `n8n-nodes-base.httpRequest` (Google Drive API + Sheets API)
- **Credential:** `googleSheetsOAuth2Api` (reused for both Drive and Sheets API)
- **Drive endpoint:** `POST https://www.googleapis.com/drive/v3/files`
- **Sheets endpoint:** `POST https://sheets.googleapis.com/v4/spreadsheets/{id}:batchUpdate`

### WF10 — Sheet Sync
- **Trigger:** `scheduleTrigger` every 10 minutes
- **Read node:** `n8n-nodes-base.googleSheets` (operation: `read`)
- **Document ID:** Dynamic — queried from Supabase `daily_intake_sheets` table
- **Write-back node:** `n8n-nodes-base.googleSheets` (operation: `update`, match by `phone_number`)
- **Upsert node:** `n8n-nodes-base.postgres` (operation: `upsert`, conflict: `phone`)

### WF1–WF5 (Reminder Workflows)
- **Read patients:** `n8n-nodes-base.postgres` (operation: `executeQuery`, SELECT from `public.patients`)
- **Update patient:** `n8n-nodes-base.postgres` (operation: `update`, match by `id`)
- **Log messages:** `n8n-nodes-base.postgres` (operation: `insert`, table: `public.message_logs`)

### WF6 — Feedback Listener
- **Find patient by phone:** `n8n-nodes-base.postgres` (operation: `executeQuery`, SELECT WHERE phone = ?)
- **Update response:** `n8n-nodes-base.postgres` (operation: `update`, match by `phone`)
